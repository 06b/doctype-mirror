APIs and Tools

The following is a list of APIs and tools useful for implementing BiDi support.

Template Systems

  • Closure Templates

GWT

  • GWT automatically mirrors page layout for RTL locales (try the Showcase in Arabic)
  • Use com.google.gwt.i18n.client.LocaleInfo's getCurrentLocale().isRTL() to determine whether the current locale is RTL.
  • com.google.gwt.resources.client.CssResource mirrors the CSS in an RTL locale.
  • com.google.gwt.resources.client.ImageResource flips images with the @ImageOptions(flipRtl = true) annotation in an RTL locale.
  • Several widgets (including TextBox, Label, InlineLabel, etc.) implement the com.google.gwt.i18n.client.HasDirection interface, which allows controlling their directionality.

JavaScript

  • Closure's goog.i18n.BidiFormatter does directionality estimation, declaration, and reset, as well as style mirroring.
  • Closure's goog.i18n.bidi has functions for directionality estimation as well as style mirroring and other useful stuff.
  • Closure's goog.ui.BidiInput is a text input widget that includes directionality auto-detection, adjusting its directionality as the user enters text.
  • Many of Closure's goog.ui widgets support BiDi, including: Button, Component, Container, BasicMenu, Popup, SubMenu, Toolbar, ToolbarButton.

Tools

  • Use CSSJanus to mirror CSS files developed for LTR pages for use in RTL pages.
